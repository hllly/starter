# 线索页设计方案 v1

## 一、整体设计目标

这版线索页不再是"信息展示页"，而是：

**一个按处理状态切换语境、按决策顺序组织信息的线索处理工作台。**

核心阅读顺序固定为：

1. 价值
2. 类型
3. URL
4. 公司名
5. 地区 / 品类
6. 联系方式 / 规模 / 来源 / 入选原因
7. 当前状态 / 反馈处理

> **注意：** "价值 / 类型 / URL"提到了最核心位置，甚至比公司名更接近"判断入口"。公司名仍然要显眼，但判断逻辑上不能再只盯公司名。

---

## 二、页面整体结构

### 顶部概览区

保留四个概览卡，重新定义语义和样式：

| 概览卡 | 语义 |
|--------|------|
| 本轮线索 | 总数 |
| 高价值 | recommended tier 数量 |
| 建议联系 | contact_now + contact_if_fit |
| 已处理 | 非 new 状态数量 |

**样式规范：**

- 高度：96px
- 圆角：20px
- 背景：白色
- 边框：1px 浅灰
- 数字字号：32px，半粗
- 标签字号：15px，中灰
- 当前 tab 对应概览卡可有轻微主题描边

### Tab 区

当前激活 tab 下方**整个列表区域**都有一层"场景底色"：

- 顶部 tabs 是状态切换
- 下方列表容器背景跟着当前 tab 切换

用户一眼就知道现在自己在什么处理语境里。

---

## 三、Tab 整体底色规则

整体底色控制在**低饱和、低对比、轻雾感**区间，不做强彩色面板。

| Tab | 背景色 | 感受 | 作用 |
|-----|--------|------|------|
| 全部 | `#F7F6F2` | 中性暖灰白 | 默认浏览态 |
| 待处理 | `#F3F8F7` | 浅青灰 | 提示"有待动作" |
| 已关注 | `#EEF8F4` | 浅青绿 | 正向保留状态 |
| 已联系 | `#F1F5FA` | 浅蓝灰 | 已经推进过 |
| 暂不跟进 | `#F6F3F0` | 浅暖灰米 | 弱化、收纳 |
| 交易中 | `#F3F9F2` | 极浅绿色 | 活跃商机 |

---

## 四、列表容器设计

每个 tab 下，线索列表不直接铺在页面上，要有一个统一的"当前视图容器"。

**容器样式：**

- 背景：使用当前 tab 对应整体底色
- 圆角：24px
- 内边距：24px 24px 32px
- 与页面边缘有明显留白

让用户感觉这是一个"当前状态工作区"。

---

## 五、单张线索卡设计

每张卡片从"均匀字段卡"改成"判断型工作卡"。

**卡片外层样式：**

- 背景：白色
- 边框：1px `#E7E3DA`
- 圆角：22px
- 阴影：`0 4px 14px rgba(0,0,0,0.03)`
- 左侧状态条：6px 宽，随状态变化
- 卡片间距：20px

---

## 六、卡片三层结构

### 第一层：一级判断区

**最重要的区域。**

**左上主信息：**

- 第一行：`[价值标签]` `[客户类型标签]` `URL`
- 第二行：公司名
- 第三行：地区 | 品类

**右上辅助判断：**

- 建议动作
- 当前处理状态

### 第二层：二级字段区

做成一组**有底色、有标题、字体更大**的字段块胶囊卡：

- 联系方式
- 规模
- 来源
- 入选原因

> 不是普通灰字说明，而是有背景、有标题、有较大正文字号的字段块。

### 第三层：操作区

- 当前状态
- 反馈 / 处理按钮
- 快捷操作（可选）

## 九、二级字段块设计

**本方案重点之一。**

### 布局

4 个字段块横向排布，桌面端每行 4 个。空间不足变 2×2。

字段：联系方式 / 规模 / 来源 / 入选原因

### 字段块统一样式

| 属性 | 值 |
|------|-----|
| 背景 | `#F7F5F0` |
| 边框 | 1px `#ECE6DD` |
| 圆角 | 16px |
| 内边距 | 16px 18px |
| 最小高度 | 96px |

### 标题

| 属性 | 值 |
|------|-----|
| 字号 | 14px |
| 字重 | 600 |
| 颜色 | `#6D6A63` |

### 内容

| 属性 | 值 |
|------|-----|
| 字号 | 18px |
| 字重 | 600 |
| 行高 | 28px |
| 颜色 | `#222521` |

> 二级字段内容不能是 13px 小灰字，要成为明显的"可读信息块"。

### 各字段块建议内容

**联系方式**

优先显示：邮箱 → 联系页 → LinkedIn → 官网

不足时：`官网可访问` / `暂无邮箱` / `LinkedIn 无`

**规模**

内容：小型 / 中型 / 大型 / 未知

可选第二行小字：`估算员工数 50-200` / `多品牌集团`

**来源**

内容：行业目录 / 平台下钻 / 官网提取

**入选原因**

内容：面向用户的自然语言，如 `宠物用品相关` / `独立官网清晰` / `疑似批发买家`

> 不要一整串程序式英文，一定要改写成面向用户的语言。

## 十三、推荐页面排版

### 区域顺序

1. 顶部概览卡
2. Tabs 状态切换
3. 当前状态工作区容器
4. 线索卡列表

### 状态工作区容器示例

以"已联系"tab 为例：

- 整个列表区域底色是浅蓝灰 `#F1F5FA`
- 每张卡片是白底
- 卡片左侧状态条是蓝灰 `#7A9AC3`
- 当前状态胶囊也是蓝灰

用户会明确知道：**现在自己处在"已联系视图"**

## 十四、给前端的具体设计原则

### 原则 1

将每个 tab 下的线索列表包裹在一个对应状态场景底色容器中，而不是只改变 tab 按钮颜色。

### 原则 2

将每条线索卡片改为白底独立卡片 + 左侧状态色条，强化卡片边界。

### 原则 3

将"价值 / 类型 / URL"重构为卡片一级判断组：

- 价值最高强调
- 类型次强调
- URL 提升为一级文本
- 公司名仍大，但不再是唯一焦点

### 原则 4

将"联系方式 / 规模 / 来源 / 入选原因"做成有底色的二级字段块，并将字段内容字号提升到 18px 左右。

### 原则 5

将"地区 / 品类"降为二级识别信息，不与一级判断组竞争。
